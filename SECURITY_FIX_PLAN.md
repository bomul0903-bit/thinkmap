# 🛡️ 퍼플에그 프로젝트 보안 개선 계획

본 문서는 토이 프로젝트를 외부 인터넷에 공개하기 전, 최소한의 보안성을 확보하기 위해 반드시 수행해야 할 개선 작업들을 우선순위별로 정리한 로드맵입니다.

---

## 🚨 1순위: 즉시 수정 필요 (치명적 위험)

### 1.1 관리자 계정 보안 강화
*   **문제점:** 소스 코드 내에 비밀번호(`purpleegg2024!`)가 하드코딩되어 있으며, 단순 문자열 비교로 인증을 수행함.
*   **해결책:** 
    *   `passlib`을 이용한 **비밀번호 해시화(Bcrypt)** 도입.
    *   소스 코드에서 하드코딩된 아이디/비밀번호 삭제 및 `.env` 파일로 분리.
    *   DB에 관리자 테이블을 생성하여 관리하거나, 환경변수에 해시된 값을 저장하여 비교.

### 1.2 JWT 시크릿 키 은닉 및 변경
*   **문제점:** 기본값(`change-this-secret-key...`)을 그대로 사용하여 토큰 위조가 가능함.
*   **해결책:** 
    *   강력한 무작위 문자열로 `SECRET_KEY` 생성 및 `.env`에 보관.
    *   운영 환경 배포 시 절대 소스 코드에 키가 포함되지 않도록 확인.

---

## ⚠️ 2순위: 데이터 보호 (개인정보 및 인증 탈취 방지)

### 2.1 인증 토큰 저장 방식 변경 (localStorage → Cookie)
*   **문제점:** `localStorage`는 XSS 공격 시 자바스크립트로 접근이 가능하여 토큰이 쉽게 탈취됨.
*   **해결책:** 
    *   토큰 발급 시 `HttpOnly`, `Secure`, `SameSite=Lax` 속성이 부여된 **쿠키**를 사용하도록 변경.
    *   프론트엔드 API 호출 시 쿠키를 자동으로 전송하도록 설정.

### 2.2 CORS 정책 정교화
*   **문제점:** 로컬 개발용 포트(`3000`, `3001`)가 모두 허용되어 있음.
*   **해결책:** 
    *   `allow_origins` 리스트를 실제 서비스할 도메인만 허용하도록 화이트리스트화.
    *   환경 변수를 통해 개발/운영 환경의 오리진을 분리하여 관리.

---

## 🛡️ 3순위: 시스템 안정성 (남용 방지)

### 3.1 Rate Limiting (속도 제한) 적용
*   **문제점:** 상담 신청 API나 로그인 API에 제한이 없어 무차별 대입 공격 및 스팸 발송에 취약함.
*   **해결책:** 
    *   `slowapi` 등 라이브러리를 설치하여 IP당 요청 횟수 제한.
    *   특히 이메일 발송이 일어나는 `/api/consultations` 엔드포인트는 강력한 제한 필요.

### 3.2 개인정보 수집 동의 및 관리
*   **문제점:** 이름, 연락처 등 개인정보를 수집하지만 최소한의 안내가 부족함.
*   **해결책:** 
    *   상담 신청 폼 하단에 개인정보 수집 및 이용에 대한 간단한 동의 체크박스 추가.
    *   DB 내 민감 정보 암호화 검토 (추후 확장 시).

---

## 📋 실행 가이드

1.  **환경 변수 설정**: `.env.example` 파일을 만들어 관리해야 할 변수 목록을 정의하고, `.env`는 절대 Git에 커밋하지 않습니다.
2.  **의존성 확인**: `passlib`, `bcrypt`, `slowapi` 등 보안 관련 라이브러리가 `requirements.txt`에 포함되었는지 확인합니다.
3.  **검증**: 수정 후 관리자 로그인이 정상적으로 작동하는지, 잘못된 비밀번호 입력 시 지연이나 제한이 걸리는지 테스트합니다.
