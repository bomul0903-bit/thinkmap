# 퍼플에그 홈페이지 셋업 가이드

이 문서는 **딱 한 번만** 따라 하시면 됩니다. 처음이라도 10분 안에 끝납니다.

준비물: 구글 계정 하나

---

## 전체 구조

```
[방문자] → index.html → Google Apps Script → Google Sheets
                                              ↑
                                      [관리자가 여기서 확인]
```

상담 신청이 들어오면 구글 스프레드시트에 자동으로 저장됩니다. 관리자 페이지에서 상태 변경도 가능하고, 스프레드시트를 직접 열어서 확인해도 됩니다.

---

## 1단계 · Google 스프레드시트 만들기

1. https://sheets.google.com 접속
2. 왼쪽 위 `+ 빈 스프레드시트` 클릭
3. 제목을 `퍼플에그 상담 관리` 등으로 변경
4. **주소창의 URL에서 스프레드시트 ID를 메모**
   - URL 예시: `https://docs.google.com/spreadsheets/d/`**`1AbC...xyz`**`/edit`
   - 굵은 부분(`1AbC...xyz`)이 시트 ID입니다

---

## 2단계 · Apps Script 코드 붙여넣기

1. 방금 만든 스프레드시트에서 상단 메뉴 `확장 프로그램 → Apps Script` 클릭
2. 새 창이 열리면 기본 코드 `function myFunction() { ... }` 전체 삭제
3. `apps-script.gs` 파일을 열어서 **전체 내용 복사** → 붙여넣기
4. 왼쪽 위 플로피 디스크 모양(저장) 클릭 또는 `Ctrl + S`

---

## 3단계 · 스크립트 속성에 비밀번호 설정 (중요)

관리자 아이디·비밀번호와 시트 ID를 **코드에 하드코딩하지 않고 스크립트 속성에 저장**해야 안전합니다.

1. Apps Script 편집기 왼쪽 사이드바에서 **⚙️ 프로젝트 설정**(톱니바퀴) 클릭
2. 아래로 스크롤해서 **스크립트 속성** 섹션 찾기
3. **스크립트 속성 추가** 버튼 클릭
4. 다음 세 가지를 각각 추가:

| 속성 (Property) | 값 (Value) |
|---|---|
| `ADMIN_USER` | 원하는 관리자 아이디 (예: `admin`) |
| `ADMIN_PASS` | 강력한 비밀번호 (영문+숫자+특수문자 8자 이상) |
| `SHEET_ID` | 1단계에서 메모한 시트 ID |

(선택) 상담 신청 들어올 때 이메일 알림 받고 싶으면:

| 속성 | 값 |
|---|---|
| `NOTIFY_EMAIL` | 알림 받을 이메일 주소 |

5. **스크립트 속성 저장** 클릭

---

## 4단계 · 시트 초기화 함수 실행

1. Apps Script 편집기 상단 함수 선택 드롭다운(보통 `doPost` 또는 `doGet` 표시됨)에서 **`initSheets`** 선택
2. **▶ 실행** 버튼 클릭
3. 처음 실행하면 권한 요청 팝업 뜸 → `권한 검토` → 본인 구글 계정 선택 → `고급 → (프로젝트 이름)(으)로 이동(안전하지 않음)` → `허용`
   - "안전하지 않음"은 구글이 확인되지 않은 본인 스크립트에 붙이는 기본 문구이므로 안심하세요
4. 실행 로그에 `시트 초기화 완료!` 출력되면 성공
5. 스프레드시트로 돌아가보면 `consultations`, `schedules` 두 탭이 자동 생성됨

---

## 5단계 · 웹 앱으로 배포

1. Apps Script 편집기 오른쪽 위 **배포 → 새 배포** 클릭
2. 톱니바퀴 ⚙️ 클릭 → **웹 앱** 선택
3. 설정:
   - **설명**: `퍼플에그 API v1` (아무거나)
   - **다음 사용자로 실행**: **나 (본인 구글 계정)**
   - **액세스 권한이 있는 사용자**: **모든 사용자**
4. **배포** 클릭
5. 다시 권한 승인 팝업이 뜨면 허용
6. 완료 화면에서 **웹 앱 URL**을 복사
   - 예: `https://script.google.com/macros/s/AKfycby.../exec`

⚠️ 이 URL은 홈페이지에서 호출하는 주소입니다. 외부에 노출돼도 비밀번호 없이는 관리 기능에 접근할 수 없습니다.

---

## 6단계 · index.html에 URL 붙여넣기

1. `index.html` 파일을 텍스트 에디터로 열기
2. 상단의 `<script>` 블록에서 다음 부분 찾기:

```js
const CONFIG = {
  APPS_SCRIPT_URL: '',   // ← 여기!
  ...
};
```

3. 따옴표 안에 5단계에서 복사한 URL 붙여넣기:

```js
const CONFIG = {
  APPS_SCRIPT_URL: 'https://script.google.com/macros/s/AKfycby.../exec',
  ...
};
```

4. 저장

---

## 7단계 · 테스트

1. `index.html`을 브라우저에서 열기 (또는 웹 호스팅 서버에 업로드 후 접속)
2. `무료 상담 신청` 페이지로 이동
3. 테스트 데이터로 작성 후 신청
4. `상담 신청이 완료되었습니다!` 메시지 확인
5. Google 스프레드시트 열어 `consultations` 탭에 데이터가 들어왔는지 확인
6. 홈페이지 푸터의 `관리자` 링크 클릭 → 3단계에서 설정한 아이디/비밀번호로 로그인
7. 대시보드에 방금 신청한 내역이 보이면 성공! 🎉

## 추가로 채워야 할 곳들

소스코드 내 `[...]` 대괄호로 표시된 플레이스홀더들을 실제 정보로 교체해야 합니다.

### index.html 내부

`Ctrl + F`로 다음 단어들을 찾아서 수정:

| 찾을 문자열 | 교체할 내용 |
|---|---|
| `[대표자명]` | 학원 대표자 이름 |
| `[000-00-00000]` | 사업자등록번호 |
| `[학원 도로명 주소]` | 학원 주소 |
| `[대표 전화번호]` | 대표 전화번호 |
| `[대표 이메일]` | 대표 이메일 |
| `[책임자명]` | 개인정보 보호책임자 |
| `[담당자 연락처]` / `[담당자 이메일]` | 담당자 연락처 |
| `[시/군/구 입력]` / `[도로명 주소 입력]` | JSON-LD 구조화 데이터 내 주소 |
| `https://example.com/` | 실제 도메인 주소 (5곳) |

### 이미지 파일명 변경 권장 (선택)

한글 파일명은 배포 환경에 따라 로딩 실패 가능. 가능하면 영문으로 바꾸세요:

| 현재 | 권장 |
|---|---|
| `홈피 대문.png` | `hero-banner.png` |
| `홈피 대문@2x.png` | `hero-banner@2x.png` |
| `홈피 대문@3x.png` | `hero-banner@3x.png` |

파일명을 바꿨으면 `index.html`의 `홈피 대문.png` 관련 줄(hero 섹션)도 함께 수정.

### OG 이미지 만들기 (선택·권장)

카카오톡·페이스북에서 링크 공유 시 노출되는 썸네일 이미지(`og-image.png`, 1200×630px)를 만들어 두면 좋습니다. 만들지 않아도 사이트는 동작합니다.

## 배포 (호스팅)

현재 `index.html`은 단일 파일이므로 간단하게 호스팅할 수 있습니다.

### 무료 옵션 추천

1. **Netlify** – 이 파일과 이미지들을 드래그 앤 드롭으로 올리면 자동 배포. `https://무언가.netlify.app` 주소 제공. 커스텀 도메인 연결도 가능
2. **Vercel** – 비슷한 수준으로 쉬움
3. **GitHub Pages** – 깃허브 계정이 있다면 무료

### 유료 도메인을 쓰고 싶으면

- 가비아, 호스팅KR 등에서 `.com` 또는 `.co.kr` 도메인 구매 (연 1~2만 원)
- 위 무료 호스팅 서비스에서 도메인 연결 설정
- 구매한 도메인이 확정되면 `index.html` 내 `https://example.com/` 다섯 곳을 실제 도메인으로 교체

## 코드 수정 후 반영

**중요:** Apps Script 코드를 수정한 경우에는 **단순 저장만으로는 반영되지 않습니다**. 반드시 `배포 → 배포 관리 → ✏️(편집) → 새 버전 → 배포` 순서로 재배포해야 최신 코드가 반영됩니다. (URL은 그대로 유지됩니다.)

## 문제 해결

### 상담 신청 시 "서버가 설정되지 않았습니다" 오류
→ `index.html`의 `CONFIG.APPS_SCRIPT_URL`이 비어 있습니다. 6단계를 확인하세요.

### 상담 신청 시 네트워크 오류
→ 5단계에서 **액세스 권한이 있는 사용자**가 `모든 사용자`로 설정됐는지 확인하세요.

### 관리자 로그인 시 "기본 비밀번호가 변경되지 않았습니다"
→ 3단계의 스크립트 속성에 `ADMIN_PASS`를 실제로 저장하지 않았습니다. 기본값(`CHANGE_ME_PLEASE_!`)이 감지되면 로그인을 거부합니다.

### 시트에 데이터가 저장되지 않음
→ 4단계의 `initSheets` 실행을 건너뛰었거나, 5단계에서 **다음 사용자로 실행**이 본인 계정이 아닐 수 있습니다. 재배포하세요.

### 스크립트 속성을 수정했는데도 변경 안 됨
→ Apps Script 코드 수정과 마찬가지로 **재배포** 필요.

---

## 관리자 비밀번호 분실

1. Apps Script 편집기 → ⚙️ 프로젝트 설정 → 스크립트 속성
2. `ADMIN_PASS` 항목 수정 → 저장
3. 재배포
